    <hr> element[#]

    Now for everyone's most favourite element, <hr>! By default, it will scale the ornamental separator if the width of the element is smaller than 900px, else it will repeat the pattern. This is shown below.

    If you want to always use the scaled ornamental separator, use the "pattern-stretch" class. Though note that it has a "min-width" of 280px, and a "max-width" of 1064px.

    If you want to always use the repeating pattern, use the "pattern-repeat" class.

    Toggle switches[#]

    Checkboxes must be inside labels in order to be styled, and must be the first or last child element


    Note that without the <span> element, the checkbox would always be at the right, since CSS only knows where an element is relative to other elements, not where they are relative to any text.

    "display: inline-flex;" is used to center the text relative to the checkbox

    Same stuff applies to radio buttons






    The toggle switches have a fixed size of 24px, however, if the <input> element has the "scalable" class, the size will be determined by the font size. However, you lose the smooth transition if you do this.


    Again, this also holds true for radio buttons

    Dialog boxes[#]

    Style for dialogs pretty basic, it's similar to the "rounded-box" class. If there's an <h1> as the first element, it will act as the dialog box's title. The max-width is also set to 550px or the width of the viewport, whichevers smaller. Paragraphs also have half the top/bottom margin.

    Note how the <form> element in the dialog has a "button-row" and "euqal-width" class. By default anything with the "button-row" class is a flexbox with "justify-content: space-around". But if it also has an "equal-width" class, all the children will (oppertunistically) be of equal width.

    Fancy buttons[#]

    Circular buttons! These are always at a 1:1 ratio. Note that no effort was put into danger/warning/success variants beyond changing the colour of the gem.

    The fancy buttons, which can be created using the "fantasy" and "fantasy-ornamental", do not come in differing heights, but will themselves shrink by approx 76% if the screen width is smaller than 600px. You can prevent this by adding the "no-scale" class on the buttons.

    These are the fantasy buttons

    Note that if you're going to style a checkbox as a fantasy button, the label must contain an element.

    These are the fantasy-ornamental buttons

    ...which can also be check boxes (assuming you have that span)

    Fancy Input Group[#]

    Sometimes we group inputs together, here's what that looks like. By default it's a flexbox that aligns things to the center and justifies to space-between. It also lights up if any inputs inside have focus, and will use a disabled style if all of its inner inputs are disabled.

    The flex-direction switches to "column" if the first element is an <input> and has less than 2 siblings, with an alignment of "stretch" and a justification of "space-between"

    There also exists an "input-group-inner-stack" class, here's this example that's laid out in a very arbitrary way. Note that when using the "number" input type, the sans-serif font is used.

    And you can use the "small" class
